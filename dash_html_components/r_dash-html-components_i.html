Reference & Documentation

An I component. I is a wrapper for the <i> HTML5 element. For detailed attribute info see: https://developer.mozilla.org/en-US/docs/Web/HTML/Element/i

Our recommended IDE for writing Dash apps is Dash Enterprise's Data Science Workspaces, which has typeahead support for Dash Component Properties. Find out if your company is using Dash Enterprise.

children (unnamed list of or a singular dash component, character or numeric; optional): The children of this component.

id (character; optional): The ID of this component, used to identify dash components in callbacks. The ID needs to be unique across all of the components in an app.

n_clicks (numeric; default 0): An integer that represents the number of times that this element has been clicked on.

n_clicks_timestamp (numeric; default -1): An integer that represents the time (in ms since 1970) at which n_clicks changed. This can be used to tell which button was changed most recently.

disable_n_clicks (logical; optional): When True, this will disable the n_clicks prop. Use this to remove event listeners that may interfere with screen readers.

key (character; optional): A unique identifier for the component, used to improve performance by React.js while rendering components See https://reactjs.org/docs/lists-and-keys.html for more info.

accessKey (character; optional): Keyboard shortcut to activate or add focus to the element.

className (character; optional): Often used with CSS to style elements with common properties.

contentEditable (character; optional): Indicates whether the element's content is editable.

dir (character; optional): Defines the text direction. Allowed values are ltr (Left-To-Right) or rtl (Right-To-Left).

draggable (character; optional): Defines whether the element can be dragged.

hidden (a value equal to: 'hidden' or 'HIDDEN' | logical; optional): Prevents rendering of given element, while keeping child elements, e.g. script elements, active.

lang (character; optional): Defines the language used in the element.

role (character; optional): Defines an explicit role for an element for use by assistive technologies.

spellCheck (character; optional): Indicates whether spell checking is allowed for the element.

style (named list; optional): Defines CSS styles which will override styles previously set.

tabIndex (character | numeric; optional): Overrides the browser's default tab order and follows the one specified instead.

title (character; optional): Text to be displayed in a tooltip when hovering over the element.

loading_state (named list; optional): Object that holds the loading state object coming from dash-renderer.

loading_state is a named list with keys:

  • component_name (character; optional): Holds the name of the component that is loading.

  • is_loading (logical; optional): Determines if the component is loading or not.

  • prop_name (character; optional): Holds which property is loading.
